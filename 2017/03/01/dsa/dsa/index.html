俗称为 链表 （link）

  • 元素（element）：包含各种数据的对象
  • 结点（node）：包含元素（element）和指向下一个结点（node）的指针（java中为引用）
  • 头结点：链表中的第一个结点（包含的元素一般为空而不是第一个元素）
  • 尾结点：链表中的最后一个结点
  • 元结点：链表中包含第一个元素的结点（当头结点包含第一个元素时头结点等于元结点）
  • 头指针：指向链表的头部（head），即指向头结点
  • 尾指针：指向链表的尾部（tail），即指向尾结点

头结点的作用

  • 简化操作：无头结点时添加和删除元结点需要特殊处理（更新head指针），有头结点时不需要

逻辑结构

  • 可选（optional）：包含一个元素的容器（container）结构
  • 合集（collection）：包含多个元素的容器（container）结构
  • 集合（set）：无序的合集（collection）
  • 列表（list）：有序的合集（collection）
    • 队列（queue）：只支持尾部插入和头部删除的列表（list）
    • 栈列（stack）：只支持头部插入和头部删除的列表（list）
  • 字典（dict）：支持键和值快速关联的映射（mapping）结构
  • 堆（heap）：父结点比子结点大或小（兄弟结点无大小关系）的树形（tree）结构
  • 跳表（skiplist）：多层链表组合的形成的表格（table）结构

优先级队列实现：堆
可排序字典实现：红黑树或者跳表

线性结构

  • 数据结构和算法之线性表详解
  • 数据结构和算法之队列和栈列详解

零个或多个数据元素的有序合集（collection），俗称为 线性表 （sequence table）

包含如下几种分类

  • 列表（list）：可以在任意位置插入（add）和删除（remove）
  • 队列（queue）：只能在尾部（rear）插入（enqueue）和头部（front）删除（dequeue）
  • 栈列（stack）：只能在顶部（top）插入（push）和删除（pop）
