DML（Data Manipulation Language）

  • es之post和put的区别入门
  • es之post和put的区别详解
  • es之各种insert的区别1
  • es之各种insert的区别2
  • es之各种insert的区别3
  • es之脚本操作
  • es之批量操作
  • insert：插入
    • insertOrReject：不存在就插入，存在时就拒绝
    • insertOrIgnore：不存在就插入，存在时就忽略
    • insertOrUpdate：不存在就插入，存在时就更新
    • insertOrReplace：不存在就插入，存在时就替换
  • update（modify）：（部分）更新（修改）
    • updateOrReject：存在时就更新，不存在就拒绝（不常见）
    • updateOrIgnore：存在时就更新，不存在就忽略
    • updateOrInsert：存在时就更新，不存在就插入（等同于insertOrUpdate）
  • replace（modify）：（整体）替换（修改）
    • replaceOrReject：存在时就替换，不存在就拒绝（不常见）
    • replaceOrIgnore：存在时就替换，不存在就忽略
    • replaceOrInsert：存在时就替换，不存在就插入（等同于insertOrReplace）

总结如下：

  1. 不指定id时，post会创建新的，put操作不合法
  2. 指定id时，如果id不存在，post会创建，put不会创建，但是put可以通过接口/_create/<_id>实现强制创建
  3. 指定id时，如果id存在，post和put都是整体更新，但是post可以通过末端/_update实现部分更新
  4. post时可以通过末端/<_id>/_create或参数op_type=create控制id冲突时拒绝插入
  5. put时可以通过末端/<_id>/_create或参数op_type=create控制id不存在时拒绝更新
  6. post时可以通过末端/_update实现部分更新
  7. put时可以通过接口/_create/<_id>控制不存在时强制创建

单个插入、批量插入

insertOrReject：es index（post /<_id>/_create） op
insertOrIgnore：es index（post /<_id>/_update_by_query） op with id condition
insertOrUpdate: es index（post /<_id>/_update） op with upsert param
insertOrReplace：es index（post） op || es index（put /_create/<_id>） op
