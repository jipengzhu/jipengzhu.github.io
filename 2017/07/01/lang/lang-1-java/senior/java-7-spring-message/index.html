系统之间进行数据通信有两种方式

  • 事件总线（EventBus）：传递的是 事件 ，基于 发布订阅模式 实现（事件被订阅者 共享 订阅）
  • 消息队列（MessageQueue）：传递的是 消息 ，基于 生产消费模式 实现（消息被消费者 分工 消费）

Publish-Subscribe

发布订阅模式和观察者模式的区别

  • 发布订阅模式和观察者模式的区别
  • 观察者模式：被观察者和观察者直接连接
  • 发布订阅模式：发布者和订阅者通过broker连接

Producer-Consumer

生产消费模式和发布订阅模式的区别

  • 生产消费模式和发布订阅模式的区别
  • 发布订阅模式：传递的是事件，事件被订阅者共享订阅
  • 生产消费模式：传递的是消息，消息被消费者分工消费
