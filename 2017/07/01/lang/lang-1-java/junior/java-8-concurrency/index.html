线程的状态

  • Java线程的状态

NEW、READY、RUNNING、WAITING、TIMED_WAITING、BLOCKED、TERMINATED

  • NEW（新建）：创建线程之后
  • READY（就绪）：调用Thread.start后
  • RUNNING（运行）：获取cpu（还有lock）资源成功后
  • WAITING（等待）：调用Object.wait、Thread.join、LockSupport.park方法后
  • TIMED_WAITING（超时等待）：调用Object.wait、Thread.join、Thread.sleep、LockSupport.park方法后
  • BLOCKED（阻塞）：获取lock失败后
  • TERMINATED（死亡）：结束线程后

ThreadLocalMap为什么不用Thread做key

  • ThreadLocalMap为什么不用Thread做key
  • HashMap线程不安全导致的死循环和数据丢失
  1. 如果ThreadLocalMap是ThreadLocal的成员变量，并且用Thread做key，那么ThreadLocalMap就会被多个线程访问，会出现丢失数据的线程安全问题（可以参考hashmap的线程安全问题）
  2. 如果ThreadLocalMap是Thread的成员变量，并且用ThreadLocal做key，那么ThreadLocalMap就是线程私有变量，只会被持有的线程访问，不会出现丢失数据的线程安全问题

锁的分类

  • 是否锁住资源：悲观锁（锁住资源）、乐观锁（不锁住资源）
  • 是否独占资源：排他锁（独占资源）、共享锁（共享资源）、读写锁（读共享和写独占）
  • 是否阻塞线程：互斥锁（阻塞）、自旋锁（不阻塞）
  • 是否支持重入：可重入锁（同一个线程可以多次获取锁）
  • 是否支持中断：可中断锁（中断后会抛出InterruptedException）
  • 是否支持公平：公平锁（排队且先进先出）、非公平锁（先插队且失败后再排队）
  • 是否支持升级：无锁、偏向锁、轻量级锁、重量级锁

synchronized和volatile的区别

  • synchronized和volatile的区别
  • synchronized可以修饰类、字段、方法，volatile只能修饰字段

synchronized和Lock的区别

  • synchronized和Lock的区别

位置：

  • synchronized是java关键字，Lock是java类

操作：

  • synchronized会自动释放锁，Lock需要手动释放
  • synchronized不能设置等待超时时间，Lock可以设置等待超时时间

状态：

  • synchronized无法判断是否获取了锁，Lock可以判断是否获取了锁

特性：

  • synchronized不支持中断，Lock可支持中断
  • synchronized不支持公平锁，Lock可支持公平锁（也支持非公平）

实现：

  • synchronized基于操作系统指令实现，Lock基于jvm实现

任务的调度方式
  • 任务的控制方式
  • 顺序（sequential）：主体是任务（task），后一个任务必须等待前一个任务完成后才执行
  • 并发（concurrent）：主体是任务（task），后一个任务不必等待前一个任务完成后才执行
  • 串行（serial）：主体是执行器（cpu、thread），执行器只能依次执行每个任务
  • 并行（parallel）：主体是执行器（cpu、thread），执行器可以同时执行多个任务
并发和并行的区别
  • 并发和并行的区别
  • 并发和并行图解
  • 并发（Concurrency）：主体是任务（task），指多个任务在某个时段被执行器分时交替执行
  • 并行（Parallelism）：主体是执行器（cpu、thread），指执行器在某个时刻对多个任务同时独立执行

并发关注的是任务，更接近人的关注点，并行关注的是执行器（源），因此并行算是并发的一种方式

线程池的状态变化
  • 线程池的状态详解

RUNNING、SHUTDOWN、STOP、TIDYING、TERMINATED

  • RUNNING（运行）：执行submit()或execute()后
  • SHUTDOWN（关闭）：执行shutdown()后，不接收新任务，但能处理已排队的任务
  • STOP（停止）：执行shutdownNow()后，不接收新任务，不处理已排队的任务，中断当前任务
  • TIDYING（清扫）：线程池中的任务为空后
  • TERMINATED（终结）：执行terminated()后

快速创建线程池的方法
Executors.newSingleThreadExecutor

线程数固定（单个），队列容量无限大，适合串行执行的任务

Executors.newFixedThreadPool

线程数固定（多个），队列容量无限大，线程数量固定且消耗资源少，任务需要排队，适合耗时长的任务

Executors.newCachedThreadPool

线程数无限大，队列容量固定（0个），线程数量充足且有保活时间，任务不用排队，适合耗时短的任务

Executors.newScheduledThreadPool

线程数无限大，队列容量无限大，支持延时和定时任务

快速创建线程池的问题
  • Executors.newSingleThreadExecutor：队列容量无限大，容易导致内存耗尽引发OOM
  • Executors.newFixedThreadPool：队列容量无限大，容易导致内存耗尽引发OOM
  • Executors.newCachedThreadPool：线程数量无限大，容易导致内存耗尽引发OOM
  • Executors.newScheduledThreadPool：线程数量无限大，容易导致内存耗尽引发OOM
