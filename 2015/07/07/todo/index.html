并发和并行 是通过对任务（task）的切分和工作者（worker，更抽象的说法为资源 resource）的增加来减少处理任务的时间

举例
做菜：称为task，包含洗菜、切菜、炒菜三个步骤
洗菜：称为subTask1，耗时为t1
切菜：称为subTask2，耗时为t2
炒菜：称为subTask3，耗时为t3
时间:t1 < t2 < t3

为了便于理解 负责处理task的worker称为worker。负责处理subTask的worker称为subWorker。

未切分
只有一个worker
完成1个task耗时为 t1 + t2 + t3
完成n个task耗时为 n * (t1 + t2 + t3)

垂直切分（并发）
将1个task分为k个不相互独立的subTask

有1个workerGroup，负责n个task
1个workerGroup有k个worker，各自负责一个subTask
所以一共需要k个worker

完成1个task耗时为 t1 + t2 + t3

乍一看没有提高效率，但是比如subTask1的工作者worker1不必等待整个task完成就可以继续下一个task的subTask1
第一个task时subTask2需要等待subTask1，subTask3需要等待subTask2
后续的task由于t1＜ t2，subTask2执行时subTask1已经完成了，由于t2＜ t3，subTask3执行时subTask2已经完成了
所以后续的task只需要耗时 (n - 1) * t3
所以完成n个task耗时为
t1 + t2 + t3 + (n - 1) * t3 = t1 + t2 + n * t3为比切分时少，所以提高了效率

如果t2大于t3（比如需要菜需要腌制），subTask3需要等待subTask2，耗时就不好计算了，但是还是会比未切分小

水平切分（并行）
将n个task分为m个相互独立的taskGroup

有m个workerGroup，各自负责一个taskGroup
1个workerGroup有k个worker，各自负责一个subTask
所以一共需要m*k个worker

完成1个task耗时为 t1 + t2 + t3

完成n个task耗时为 n/m * (t1 + t2 + t3)

效率提高到了m倍

问题

  1. 垂直切分中subTask的依赖会导致任务暂停等待，使得效率降低，即生产者的速度低于消费者
  2. 垂直切分中worker不足也会导致任务暂停等待，使得效率降低，即worker的数量小于subTask的数量
  3. 水平切分中的资源浪费，subTask1和subTask3都有m个worker，但是subTask1的活干完后到subTask3的活干完时subTask1的worker处于空闲状态，所以subTask1要少点worker，所以subTask3要多点worker

灵活切分
有k个workerGroup，各自负责一个subTask
1个workerGroup有若干个worker
